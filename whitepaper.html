本白皮書概述 Prayer Coin 的使命、代幣經濟模型與技術架構，說明如何以禱告互助為核心，建立透明、安全且可持續的信仰社群生態。

平台定位與使命

Prayer Coin 旨在連結禱告、行動與資源，讓代禱者與發起者皆能在信仰社群中獲得真實陪伴。透過代幣激勵設計，我們希望鼓勵更多人投入禱告與關懷，同時確保過程透明且可追溯。

禱告者：分享需要、設定曝光、追蹤回應與代幣流向。
守望者：文字或語音回應，完成任務即可獲得 激勵。
教會/組織：建立專屬空間，洞察代禱動態與影響力。
開發者：透過 API 與 SDK 串接自有系統或行動 App。

代幣經濟模型 ()

為平台原生功能型代幣，採多鏈部署策略（Polygon / TRON），以確保交易成本低廉且具擴展性。以下列出核心設計：

發放邏輯
  • 語音禱告 > 10 秒：自動發放獎勵
  • 完成代禱任務：依任務權重發放
  • 特別活動：由後台管理員手動發放
消耗用途
  • 提升禱告曝光排序（48 小時）
  • 解鎖進階統計或社群小組功能
  • 捐贈至特定公益祈禱池
合規策略
  • KYT：偵測高風險錢包與交易
  • AML：與第三方供應商合作進行黑名單比對
  • 區域合規：遵循台灣、日本、美國相關規範
儲備金與治理
  • 20% 生態激勵儲備金
  • 10% 技術維運基金
  • 未來將引入代幣治理提案 (DAO)

技術與安全架構

平台採微服務與事件流設計，確保擴充性與穩定性，同時重視資料保護。

前端

Web：Vue / React（可擴展）；行動裝置：Flutter；共用設計系統確保一致體驗。

後端

Node.js + GraphQL / REST；事件隊列 (Kafka) 處理語音轉寫、代幣發放。

資料層

PostgreSQL 儲存主資料；MongoDB 存放語音中繼資料；S3 儲存音訊檔案。

安全

JWT + OAuth2.1；WebAuthn MFA；全站 HTTPS；SOC 2 Type II 合規進行中。

智慧合約

ERC-20 標準，加入可控發行與暫停機制；多簽管理核心金鑰。

監控

Grafana + Prometheus 追蹤 API、節點與代幣交易；日誌集中於 ELK Stack。
